# Solution Architecture: Integrating Live Translate into GeminiLiveDemo

This document outlines how we can seamlessly implement the **Gemini 3.5 Live Translate** (`gemini-3.5-live-translate-preview`) feature directly into the current `GeminiLiveDemo` app with minimal code changes.

---

## 1. User Interface (UI) Integration

We will add a toggle and selector inside [SettingsActivity.kt](file:///d:/New%20folder%20(2)/GeminiLiveDemo/app/src/main/java/com/example/geminimultimodalliveapi/SettingsActivity.kt):
- **Toggle**: `Enable Live Translate` (Saved in `AppPreferences` as `isTranslateModeEnabled`).
- **Dropdown (Spinner)**: `Target Language` (e.g., English `"en"`, Japanese `"ja"`, Chinese `"zh"`, Spanish `"es"`). Saved in `AppPreferences` as `translateTargetLanguage`.

When enabled, the microphone button on the main console will act as the trigger to connect to the Live Interpreter session.

---

## 2. WebSocket Client Modification

In [GeminiLiveClient.kt](file:///d:/New%20folder%20(2)/GeminiLiveDemo/app/src/main/java/com/example/geminimultimodalliveapi/network/GeminiLiveClient.kt), we will adjust the WebSocket configuration payload:

### A. Dynamic Model Targeting
```kotlin
// In GeminiLiveClient constructor/properties
private val MODEL = if (isTranslateMode) {
    "models/gemini-3.5-live-translate-preview"
} else {
    "models/gemini-3.1-flash-live-preview"
}
```

### B. Modified Setup JSON Payload
In `sendInitialSetupMessage()` inside [GeminiLiveClient.kt](file:///d:/New%20folder%20(2)/GeminiLiveDemo/app/src/main/java/com/example/geminimultimodalliveapi/network/GeminiLiveClient.kt#L288):
- If `isTranslateMode` is `true`:
  - **Skip** adding `systemInstruction` and `tools` (since the translation-specialized model does not support reasoning tools).
  - Add `translationConfig` inside `generationConfig`:
  
```kotlin
if (isTranslateMode) {
    val translationConfig = JSONObject()
    translationConfig.put("targetLanguageCode", targetLanguageCode) // e.g. "en"
    translationConfig.put("echoTargetLanguage", true)
    generationConfig.put("translationConfig", translationConfig)
}
```

---

## 3. Audio & UI Stream Integration

Because `gemini-3.5-live-translate-preview` matches the standard input/output specifications:
- **Audio Input**: Raw 16kHz PCM (already generated by our `AudioRecorder`).
- **Audio Output**: 24kHz PCM (already decoded and played by our `AudioPlayer` jitter buffer).

The returned translated text/audio will stream back over the same WebSocket, automatically triggering `onTextMessageReceived()` and `onAudioChunkReceived()` in the client listener.
- The translated text (e.g. *"Hello, how are you?"*) will automatically show up in [MainActivity.kt](file:///d:/New%20folder%20(2)/GeminiLiveDemo/app/src/main/java/com/example/geminimultimodalliveapi/MainActivity.kt)'s console log view!
